Verbindungen anlegen

Alle Felder im Verbindungs-Dialog erklärt — und wie das Laufwerk danach eingehängt wird.

Verbindungs-Dialog

Feld Beschreibung Standard
nameAnzeigename, wird als Laufwerksbezeichnung gesetzt.—
hostAdresse oder IP des SSH-Servers.—
userSSH-Benutzername auf dem Server.—
portSSH-Port.22
remote_pathDer Ordner auf dem Server, der als Laufwerk eingehängt werden soll./
drive_letterWindows-Laufwerksbuchstabe (z. B. Z:).Z:
auth_methodpassword, key oder ask.password
passwordWird lokal verschlüsselt in der SQLite-Datenbank gespeichert.—
key_pathPfad zum SSH-Key (nur bei Methode „key").—
cli_access_enabledErlaubt Zugriff aus dem CLI-Companion.false

Laufwerk einhängen

Den Toggle rechts in der Verbindungskarte klicken. Bei Erfolg wird der Toggle grün und das Laufwerk erscheint im Explorer.

Laufwerk aushängen

Nochmals auf den (nun grünen) Toggle klicken. Falls das fehlschlägt, werden automatisch Fallback-Methoden versucht.

Tipps

  • Auto-Reconnect-Mounts beim Start in den Einstellungen aktivieren, damit Laufwerke beim nächsten App-Start automatisch wieder eingehängt werden.
  • Bei „mount point in use": Einstellungen → Veraltete Mounts bereinigen.
  • Mehrere Verbindungen zum gleichen Server sind erlaubt — jeweils mit anderem Laufwerksbuchstaben.
